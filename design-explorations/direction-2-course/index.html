Why Python — and why you can't skip this

Every major AI and agent framework — LangChain, LangGraph, Pydantic, FastAPI, the Anthropic SDK, the OpenAI SDK — is written in Python. This isn't an accident. Python reads like pseudocode, which means AI researchers could express ideas as code without becoming full-time software engineers. That culture compounded over twenty years. The entire AI ecosystem lives here now.

You don't need to be a Python expert before you start. But you do need a solid grip on the six features that agent frameworks use constantly. Skip any of them and you'll spend your time debugging framework internals instead of building things.

💡
Note We're not teaching all of Python — that would take months. We're covering the 20% that appears in 80% of AI engineering code.

Type hints — the one Python feature you must learn first

Python is dynamically typed. You write x = 5 and Python figures out the type at runtime. This is great for experimentation and terrible for large systems. That's why modern Python uses type hints — annotations that tell both humans and tools what type a variable should be.

python 
# Without type hints — works, but your editor can't help you
def greet(name):
    return f"Hello, {name}"

# With type hints — same code, but now your editor + Pydantic understand it
def greet(name: str) -> str:
    return f"Hello, {name}"

# Real-world consequence: LangChain reads type hints to build tool schemas
# If you skip them, your tools silently break in unpredictable ways

Wrong way vs right way

✗ Rookie mistake
# Mutable default argument
def add(item, lst=[]):
    lst.append(item)
    return lst

add("a")  # ['a']
add("b")  # ['a','b'] ← bug!
✓ Correct
# Use None as sentinel
def add(item, lst=None):
    if lst is None:
        lst = []
    lst.append(item)
    return lst

add("a")  # ['a']
add("b")  # ['b'] ✓

Self-check

What happens if you call a function with a mutable default argument twice?

The default object is created once when the function is defined, not each time it's called. So the second call sees the modified object from the first call — causing unexpected shared state.

Why does LangChain need type hints on your @tool functions?

LangChain reads the type hints to auto-generate the JSON Schema that gets sent to the LLM. Without them, the LLM doesn't know what parameters the tool accepts, and tool calls fail silently.
