Why Python

LangChain, LangGraph, Pydantic, FastAPI, the Anthropic SDK, the OpenAI SDK — all Python. The AI ecosystem converged here because Python reads like pseudocode: researchers could write ideas as code before becoming full-time engineers. That compounded for twenty years. There is no practical alternative in 2026.

You don't need to be a Python expert. You need a grip on the features agent frameworks use constantly. Skip any of them and you'll debug framework internals instead of shipping.

Scope note
This module covers the 20% of Python that appears in 80% of AI engineering code. Not all of Python — that's a different course.

Environment setup

On Windows, python may open the Microsoft Store instead of running. Use python3 or install Python from python.org and check "Add to PATH".

Type hints

Python is dynamically typed — x = 5 and Python infers the type. Type hints add annotations that don't change runtime behavior but enable editors, Pydantic, and LangChain to read your intent.

python · type_hints.py 
# Untyped — works, editor can't help you
def greet(name):
    return f"Hello, {name}"

# Typed — LangChain reads this to build tool schemas
def greet(name: str) -> str:
    return f"Hello, {name}"

# Common complex types
from typing import Optional, List, Dict

def search(
    query: str,
    top_k: int = 5,
    filters: Optional[Dict[str, str]] = None
) -> List[str]:
    ...

Decorators

A function that wraps another function to add behaviour. Every agent framework uses them. @tool in LangChain, @app.get in FastAPI, @dataclass in stdlib — once you build one from scratch, all of these click.

python · decorator_from_scratch.py 
import functools

def log_calls(func):
    @functools.wraps(func)          # preserves func.__name__
    def wrapper(*args, **kwargs):
        print(f"→ {func.__name__}({args}, {kwargs})")
        result = func(*args, **kwargs)
        print(f"← {result}")
        return result
    return wrapper

@log_calls
def add(a: int, b: int) -> int:
    return a + b

add(2, 3)
# → add((2, 3), {})
# ← 5

Common pitfalls

Mutable default argument
def f(x, lst=[]): — the list is created once at definition time, not per call. Fix: use lst=None and create a fresh list inside the function body.
Missing @functools.wraps in decorators
Without it, func.__name__ and func.__doc__ become those of wrapper. LangChain uses the docstring as the tool description — skip wraps and your tool description disappears.
Missing type hints on @tool functions
LangChain builds the JSON schema it sends to the LLM from your type hints. No hints = no schema = the LLM doesn't know what the tool accepts.

Self-check

What does @functools.wraps(func) do and why does it matter?

It copies __name__, __doc__, and other metadata from the wrapped function to the wrapper. Without it, introspection tools and frameworks (like LangChain) that read those attributes will see the wrapper's name instead of your function's name.

Name two Python stdlib types you'd use to type-hint a function that returns a list of strings or None.

Optional[List[str]] from typing, or in Python 3.10+: list[str] | None.

Why does from typing import Optional matter less in Python 3.10+?

Python 3.10 added the X | Y union syntax natively, so you can write str | None instead of Optional[str] without any import.
