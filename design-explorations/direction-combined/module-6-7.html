Long-Term Memory

This is the Phase 6 capstone. Short-term memory (6.3) handled the current session. Episodic memory (6.5) stored facts from past sessions. Long-term memory ties it all together — a structured knowledge store that grows over months, prioritises what matters based on recency and access frequency, decays what doesn't, and surfaces the right memories at the right moment. This module covers the complete architecture: a typed memory schema, importance scoring, temporal decay, a unified recall pipeline, and the full memory lifecycle from write to archive to delete.

~28 min read 5 code examples Last updated 2026-05-07
Key Takeaways
  • Long-term memory is not a flat list — it has types (preference, fact, goal, constraint, relationship) and each type has different retrieval priority and decay behaviour
  • Memory importance scoring combines recency, access frequency, and explicit importance labels — high-importance memories survive temporal decay longer
  • Temporal decay (forgetting) is a feature, not a bug — a preference stated 18 months ago may no longer be true. Memories with no recent access should decay toward archival
  • The recall pipeline is the critical path — it must run fast (under 100 ms) because it blocks every user turn. Use ANN (approximate nearest-neighbour) search, not exact nearest-neighbour
  • Long-term memory systems require operational tooling — a management UI for users to view/edit/delete memories, and a monitoring dashboard to catch memory quality regression

Typed memory schema #

A flat "store any string" approach fails at scale — you can't decide decay rates, access patterns, or retrieval priority without knowing what type of memory a record is. Use a typed schema:

pythonmemory_schema.py — typed long-term memory model
from dataclasses import dataclass, field
from datetime import datetime
from enum import Enum
from typing import Literal
import uuid

class MemoryType(Enum):
    PREFERENCE   = "preference"    # "prefers TypeScript", "dislikes verbose code"
    FACT         = "fact"          # "works at Haiilo", "located in Hamburg"
    GOAL         = "goal"          # "building a SaaS for HR teams"
    CONSTRAINT   = "constraint"    # "can't use AWS, company policy is Azure"
    RELATIONSHIP = "relationship"  # "reports to Jana", "collaborates with Deepu"
    SKILL        = "skill"         # "expert in Salesforce Apex", "learning React"


@dataclass
class Memory:
    id:           str          = field(default_factory=lambda: str(uuid.uuid4()))
    user_id:      str          = ""
    content:      str          = ""
    memory_type:  MemoryType   = MemoryType.FACT
    importance:   float        = 0.5   # 0.0 – 1.0, set at extraction time
    access_count: int          = 0     # incremented each time memory is recalled
    created_at:   datetime     = field(default_factory=datetime.utcnow)
    last_accessed:datetime     = field(default_factory=datetime.utcnow)
    last_updated: datetime     = field(default_factory=datetime.utcnow)
    source:       str          = ""    # session ID the memory was extracted from
    confidence:   float        = 1.0  # how certain the extraction was
    archived:     bool         = False

    def score(self, query_similarity: float, now: datetime | None = None) -> float:
        """Combined relevance score for re-ranking retrieved memories."""
        if now is None:
            now = datetime.utcnow()
        days_old = (now - self.last_accessed).days
        # Temporal decay: half-life of 30 days for importance=0.5 memories
        decay = 0.5 ** (days_old / (30 / self.importance))
        frequency_boost = min(0.1 * self.access_count, 0.3)
        return (0.5 * query_similarity) + (0.3 * decay) + (0.2 * self.importance) + frequency_boost

The recall pipeline #

The recall pipeline runs on every user turn. It must be fast. The two-stage approach — ANN search for candidates, then importance-weighted re-ranking — gives you both speed and quality:

pythonrecall.py — fast two-stage recall with importance re-ranking
import chromadb
from chromadb.utils.embedding_functions import OpenAIEmbeddingFunction
from datetime import datetime

embed_fn = OpenAIEmbeddingFunction(
    api_key="YOUR_KEY", model_name="text-embedding-3-small"
)
db = chromadb.PersistentClient(path="./long_term_memory")
col = db.get_or_create_collection("memories", embedding_function=embed_fn)


def recall(
    user_id: str,
    query: str,
    k: int = 6,
    candidate_k: int = 20,    # retrieve more candidates, re-rank, return top k
    min_importance: float = 0.2,
) -> list[Memory]:
    # Stage 1: ANN search — fast approximate nearest-neighbour
    results = col.query(
        query_texts=[query],
        n_results=candidate_k,
        where={"user_id": user_id, "archived": False},
        include=["documents", "distances", "metadatas"],
    )

    now = datetime.utcnow()
    candidates = []
    for doc, dist, meta in zip(
        results["documents"][0],
        results["distances"][0],
        results["metadatas"][0],
    ):
        mem = _from_meta(doc, meta)
        if mem.importance < min_importance:
            continue
        similarity = 1.0 - dist  # convert distance to similarity
        final_score = mem.score(similarity, now)
        candidates.append((final_score, mem))

    # Stage 2: Re-rank by combined score and return top k
    candidates.sort(key=lambda x: -x[0])
    recalled = [m for _, m in candidates[:k]]

    # Update access stats in background (don't block the response)
    for m in recalled:
        m.access_count += 1
        m.last_accessed = now
        _update_meta(m)

    return recalled


def format_for_context(memories: list[Memory]) -> str:
    if not memories:
        return ""
    lines = [f"[{m.memory_type.value}] {m.content}" for m in memories]
    return "<long_term_memory>\n" + "\n".join(lines) + "\n</long_term_memory>"

pythonmemory_maintenance.py — decay, archival, and contradiction detection
from datetime import datetime, timedelta

# Run as a scheduled job — daily or weekly
def decay_and_archive(user_id: str, archive_threshold: float = 0.1) -> int:
    """Archive memories whose combined decay score drops below threshold."""
    results = col.get(where={"user_id": user_id, "archived": False})
    archived_count = 0
    now = datetime.utcnow()
    for id_, meta in zip(results["ids"], results["metadatas"]):
        mem = _from_meta("", meta)
        time_score = mem.score(query_similarity=0.0, now=now)
        if time_score < archive_threshold:
            col.update(ids=[id_], metadatas=[{**meta, "archived": True}])
            archived_count += 1
    return archived_count


def detect_contradiction(user_id: str, new_memory: str) -> str | None:
    """Check if a new memory contradicts an existing one. Returns conflicting memory id or None."""
    candidates = recall(user_id=user_id, query=new_memory, k=3)
    if not candidates:
        return None
    prompt = f"""Does the NEW fact contradict any of the EXISTING facts?
Answer YES or NO only. If YES, output the contradicted fact verbatim.

NEW: {new_memory}

EXISTING:
""" + "\n".join(f"- {m.content}" for m in candidates)
    resp = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}],
        max_tokens=50,
    )
    answer = resp.choices[0].message.content.strip()
    if answer.startswith("YES"):
        return answer  # calling code should archive the contradicted memory
    return None

Putting it all together — the full memory agent #

pythonfull_memory_agent.py — agent with the complete Phase 6 memory stack
async def memory_agent_turn(
    user_id: str,
    user_query: str,
    session_buffer: TokenBudgetBuffer,   # Module 6.3
    cache: SemanticCache,                  # Module 6.4
) -> str:

    # ── Layer 1: Semantic cache (fastest path — no LLM call) ──────────────
    cached = cache.lookup(user_query)
    if cached:
        return cached

    # ── Layer 2: Recall long-term memories (async ANN search) ─────────────
    memories = recall(user_id=user_id, query=user_query, k=5)
    memory_block = format_for_context(memories)

    # ── Layer 3: Build context with structure (Module 6.2) ────────────────
    system = (
        "You are a helpful personal assistant.\n"
        "Use the facts in <long_term_memory> to personalise your response.\n"
        "Do not reveal you have stored memories unless asked."
    )
    if memory_block:
        system += f"\n\n{memory_block}"

    # ── Layer 4: Assemble messages with trimmed history (Module 6.3) ──────
    messages = session_buffer.get_messages(system, user_query)

    # ── Layer 5: Call LLM ─────────────────────────────────────────────────
    resp = await client.chat.completions.acreate(
        model="gpt-4o", messages=messages
    )
    answer = resp.choices[0].message.content

    # ── Layer 6: Update session buffer and cache ──────────────────────────
    session_buffer.add_turn(user_msg=user_query, assistant_msg=answer)
    cache.store(user_query, answer)

    return answer


# ── Session end handler ───────────────────────────────────────────────
async def on_session_end(user_id: str, session_buffer: TokenBudgetBuffer) -> None:
    # Extract and store new long-term memories (Module 6.5)
    await asyncio.to_thread(on_session_end_sync, user_id, session_buffer._history)
