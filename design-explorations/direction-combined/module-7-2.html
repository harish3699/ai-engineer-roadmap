LangGraph Fundamentals

LangGraph is a library for building stateful, multi-step agent workflows as directed graphs. Each node is a function that reads from and writes to a shared typed state object. Edges define control flow between nodes — they can be static (always go from A to B) or conditional (go to B or C based on the current state). This module covers every primitive you need: StateGraph, TypedDict state, reducers, add_node, add_edge, add_conditional_edges, START/END, and the compile + invoke pattern.

~25 min read 4 code examples Last updated 2026-05-07
Key Takeaways
  • A LangGraph graph is a StateGraph — nodes read from and write to a shared TypedDict state; edges define control flow
  • Reducers define how a field in state is updated when multiple nodes write to it — the default is last-write-wins; operator.add accumulates lists
  • Conditional edges take a function (state) → str that returns the name of the next node — this is how routing decisions live in the graph
  • Cycles (loops) are native to LangGraph — a ReAct agent is just a node that loops back to itself until a termination condition fires
  • Always define a clear termination condition — a missing END edge is the most common cause of infinite loops in LangGraph

The mental model #

LangGraph is fundamentally a state machine where each state transition is an LLM call or a function. The state is a shared dictionary that all nodes can read and write. Edges are the arrows on the whiteboard — they say "after node A finishes, go to node B." Conditional edges say "after node A, look at the state and decide which node to go to next."

LangGraph Primitives
Node
A Python function that takes state and returns a dict of state updates. Can call an LLM, a tool, or any logic.
Edge
Connects two nodes. Static edge: always goes A→B. Conditional edge: calls a router function to decide next node.
State
A TypedDict that persists across all nodes. Reducers define merge behaviour when nodes write to the same field.

Your first graph #

pythonfirst_graph.py — minimal StateGraph with two nodes and a conditional edge
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
from typing import TypedDict, Annotated
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, AIMessage

# ── 1. Define state ────────────────────────────────────────────────────
class AgentState(TypedDict):
    # Annotated[list, add_messages] means: append new messages, not replace
    messages: Annotated[list, add_messages]
    next_node: str
    analysis: str

llm = ChatOpenAI(model="gpt-4o-mini")

# ── 2. Define nodes ────────────────────────────────────────────────────
def analyse_node(state: AgentState) -> dict:
    """Analyse the user request and decide which worker to route to."""
    response = llm.invoke([
        *state["messages"],
        HumanMessage(content="Classify this request as 'code' or 'data'. Reply with one word."),
    ])
    route = response.content.strip().lower()
    return {"analysis": response.content, "next_node": route}

def code_node(state: AgentState) -> dict:
    response = llm.invoke([
        HumanMessage(content="You are a code expert. " + state["messages"][-1].content),
    ])
    return {"messages": [response]}

def data_node(state: AgentState) -> dict:
    response = llm.invoke([
        HumanMessage(content="You are a data analyst. " + state["messages"][-1].content),
    ])
    return {"messages": [response]}

# ── 3. Router function ────────────────────────────────────────────────
def route_after_analysis(state: AgentState) -> str:
    return state["next_node"]  # "code" or "data"

# ── 4. Build graph ────────────────────────────────────────────────────
builder = StateGraph(AgentState)
builder.add_node("analyse", analyse_node)
builder.add_node("code", code_node)
builder.add_node("data", data_node)

builder.add_edge(START, "analyse")              # start → analyse (always)
builder.add_conditional_edges(                   # analyse → code or data
    "analyse",
    route_after_analysis,
    {"code": "code", "data": "data"},
)
builder.add_edge("code", END)                  # code → END
builder.add_edge("data", END)                  # data → END

graph = builder.compile()

# ── 5. Invoke ────────────────────────────────────────────────────────
result = graph.invoke({"messages": [HumanMessage(content="Write a Python function to sort a list")]})
print(result["messages"][-1].content)

Reducers — how state updates merge #

When a node returns a dict, LangGraph merges it into the existing state. The merge behaviour is controlled by reducers defined in the Annotated type. Without a reducer, the default is last-write-wins (replacement). The add_messages reducer appends new messages rather than replacing the list — this is the most common reducer you'll use.

pythonreducers.py — custom reducer for aggregating results
import operator
from typing import Annotated

class PipelineState(TypedDict):
    query: str                                    # last-write-wins (default)
    messages: Annotated[list, add_messages]       # append new messages
    results: Annotated[list, operator.add]        # append new items
    error_count: Annotated[int, operator.add]     # sum error counts
    final_answer: str                              # last-write-wins

# Node A writes: {"results": ["result from A"]}
# Node B writes: {"results": ["result from B"]}
# After both: state["results"] == ["result from A", "result from B"]
# — because operator.add for lists = concatenation

Cycles and termination #

LangGraph supports directed cycles — a node can route back to an earlier node (including itself). This is how ReAct loops are implemented natively. The critical requirement: there must be a path to END that the graph will eventually take, otherwise the graph runs forever.

pythoncycle_graph.py — ReAct loop as a LangGraph cycle
class ReactState(TypedDict):
    messages: Annotated[list, add_messages]
    iterations: int
    max_iterations: int

def agent_node(state: ReactState) -> dict:
    response = llm_with_tools.invoke(state["messages"])
    return {"messages": [response], "iterations": state["iterations"] + 1}

def tools_node(state: ReactState) -> dict:
    # Execute all tool calls from the last assistant message
    tool_results = execute_tools(state["messages"][-1])
    return {"messages": tool_results}

def should_continue(state: ReactState) -> str:
    last = state["messages"][-1]
    if state["iterations"] >= state["max_iterations"]:
        return "end"                    # hard stop — prevents infinite loops
    if hasattr(last, "tool_calls") and last.tool_calls:
        return "tools"                  # continue — call tools
    return "end"                        # model gave final answer

builder = StateGraph(ReactState)
builder.add_node("agent", agent_node)
builder.add_node("tools", tools_node)
builder.add_edge(START, "agent")
builder.add_conditional_edges("agent", should_continue, {"tools": "tools", "end": END})
builder.add_edge("tools", "agent")   # tools always go back to agent → the cycle
graph = builder.compile()

Streaming graph execution #

pythonstream_graph.py — streaming intermediate state updates
# stream() yields state updates after each node completes
for event in graph.stream(
    {"messages": [HumanMessage(content="Analyse this CSV and plot revenue trends")],
     "iterations": 0, "max_iterations": 10},
    stream_mode="updates",    # "updates" | "values" | "messages"
):
    for node_name, node_output in event.items():
        print(f"[{node_name}] → {list(node_output.keys())}")

# stream_mode="updates"  → yields {node_name: state_delta} after each node
# stream_mode="values"   → yields full state after each node
# stream_mode="messages" → yields individual LLM message tokens (for streaming to UI)
