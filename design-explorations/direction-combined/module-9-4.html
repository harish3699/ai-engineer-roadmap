AI-Specific Services

Every major cloud provider now has a managed AI service layer that sits above raw compute: a model catalog you invoke without managing GPU infrastructure, a vector search service, an agent hosting runtime, and managed guardrails. The primitives are the same across AWS, GCP, and Azure — the SKU names differ. This module maps the AWS services you've seen throughout this course to their equivalents on GCP Vertex AI and Azure AI Foundry, so you're not lost when you encounter a different cloud.

~16 min read 2 code examples Last updated 2026-05-07
Key Takeaways
  • AWS Bedrock: managed foundation model API — Claude, Llama, Mistral, Titan. Invoke via boto3, no GPU management. Add Guardrails and Knowledge Bases on top
  • AWS AgentCore (2026): production agent hosting runtime — managed memory, tool execution, session state, and scaling for LangGraph and other frameworks without writing the infra yourself
  • GCP equivalent: Vertex AI Model Garden (model catalog) + Agent Builder (managed agent runtime) + Matching Engine (managed vector search) + Gemini models
  • Azure equivalent: Azure AI Foundry (model catalog, formerly Azure OpenAI) + AI Agent Service + Azure AI Search (vector search) + GPT-4/Claude/Llama via marketplace
  • The pattern is the same: managed model endpoint + managed vector store + managed agent runtime + guardrails — only the boto3/google-cloud/azure-ai client differs

AWS Bedrock #

Bedrock is AWS's managed foundation model service. You invoke Claude, Llama 3, Mistral, and other models through a unified API without provisioning any GPU infrastructure. Pricing is per-token — you pay only for what you use. Bedrock also includes Knowledge Bases (managed RAG), Guardrails (module 8.2), and Agents (managed ReAct execution).

pythonbedrock_invoke.py — streaming response from Claude on Bedrock
import boto3, json

runtime = boto3.client("bedrock-runtime", region_name="us-east-1")

def stream_claude(messages: list[dict], system: str = ""):
    body = {
        "anthropic_version": "bedrock-2023-05-31",
        "max_tokens": 2048,
        "messages": messages,
    }
    if system:
        body["system"] = system

    response = runtime.invoke_model_with_response_stream(
        modelId="us.anthropic.claude-sonnet-4-6-20250219-v1:0",   # cross-region inference
        body=json.dumps(body),
        contentType="application/json",
    )
    for event in response["body"]:
        chunk = json.loads(event["chunk"]["bytes"])
        if chunk.get("type") == "content_block_delta":
            yield chunk["delta"].get("text", "")

AWS AgentCore #

AWS AgentCore (launched 2026) is a managed runtime for production agent deployments. It handles the infrastructure that every LangGraph deployment otherwise requires you to build manually: persistent session memory, tool execution with retries and timeouts, horizontal scaling of multi-agent pipelines, and built-in trace collection. You bring your LangGraph graph definition; AgentCore handles everything else.

Key capabilities: managed memory store (persists conversation state across sessions), managed tool executor (runs tool calls in isolated environments with sandboxing), and automatic scaling based on concurrent active sessions. It's the "ECS Fargate but for agents" abstraction — you don't write task definitions or autoscaling policies.

ℹ
Bedrock embeddings for your RAG pipeline Use amazon.titan-embed-text-v2:0 (1536 dimensions) when you want to keep everything within AWS and avoid a separate OpenAI API call. Pricing is $0.02 per million tokens — cheaper than OpenAI's text-embedding-3-small at $0.02/M but comparable. The model is accessible via the same boto3 InvokeModel API, so switching is one line change.
pythonbedrock_embed.py — generate embeddings with Titan on Bedrock
import boto3, json
import numpy as np

runtime = boto3.client("bedrock-runtime", region_name="us-east-1")

def embed_text(text: str) -> list[float]:
    response = runtime.invoke_model(
        modelId="amazon.titan-embed-text-v2:0",
        body=json.dumps({"inputText": text, "dimensions": 1536, "normalize": True}),
        contentType="application/json",
        accept="application/json",
    )
    body = json.loads(response["body"].read())
    return body["embedding"]    # list[float], length 1536, already normalised

def embed_batch(texts: list[str]) -> list[list[float]]:
    # Bedrock Titan doesn't have a batch endpoint — parallelise with ThreadPoolExecutor
    from concurrent.futures import ThreadPoolExecutor
    with ThreadPoolExecutor(max_workers=10) as ex:
        return list(ex.map(embed_text, texts))
