Deployment & Realtime Delivery

This is the module that turns a locally-running LangGraph agent into a production API that streams token-by-token to a chat UI. It covers Dockerising a FastAPI agent app, writing an ECS Fargate task definition, managing secrets with AWS Secrets Manager, environment promotion from dev through staging to production, and the two streaming protocols you'll use depending on whether the communication is one-way (SSE) or bidirectional (WebSockets).

~22 min read 5 code examples Last updated 2026-05-07
Key Takeaways
  • Docker: use a multi-stage build (builder stage installs deps, final stage copies only what's needed) — reduces image size by 60–80% and speeds up ECR pushes
  • Secrets Manager: never put API keys in environment variables in your task definition — fetch from Secrets Manager at container startup via the ECS secrets injection mechanism
  • Environment promotion: dev → staging → prod, with staging using production-equivalent infra (same task size, same guardrails) — staging is where you catch "works locally, breaks in prod" issues
  • SSE for token streaming: use Server-Sent Events when the server pushes tokens to the client and the client doesn't send mid-stream messages — simpler, works over HTTP/2, no upgrade required
  • WebSockets for bidirectional: use WebSockets when the user can interrupt the agent, send a tool approval, or provide mid-stream corrections — SSE is one-way and can't handle that

Dockerising a FastAPI agent #

dockerfileDockerfile — multi-stage build for a FastAPI agent
# ── Stage 1: install dependencies ────────────────────────────────────
FROM python:3.12-slim AS builder
WORKDIR /build
COPY requirements.txt .
RUN pip install --no-cache-dir --target=/build/packages -r requirements.txt

# ── Stage 2: minimal runtime image ───────────────────────────────────
FROM python:3.12-slim AS runtime
WORKDIR /app
COPY --from=builder /build/packages /app/packages
ENV PYTHONPATH=/app/packages
COPY src/ .

# Non-root user — never run as root in production
RUN addgroup --system appuser && adduser --system --ingroup appuser appuser
USER appuser

EXPOSE 8000
CMD ["python", "-m", "uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000",
     "--workers", "2", "--timeout-keep-alive", "120"]

Secrets Manager integration #

pythonsecrets_loader.py — fetch secrets at startup, not at each request
import boto3, json
from functools import cache

@cache
def get_secret(secret_name: str) -> dict:
    """Cached — fetches once per process, not per request."""
    client = boto3.client("secretsmanager", region_name="us-east-1")
    response = client.get_secret_value(SecretId=secret_name)
    return json.loads(response["SecretString"])

# Called once at FastAPI startup
from contextlib import asynccontextmanager
from fastapi import FastAPI

@asynccontextmanager
async def lifespan(app: FastAPI):
    secrets = get_secret("agent/prod-secrets")
    app.state.openai_key  = secrets["OPENAI_API_KEY"]
    app.state.db_password = secrets["DB_PASSWORD"]
    app.state.db_pool     = await create_db_pool(password=app.state.db_password)
    yield
    await app.state.db_pool.close()

app = FastAPI(lifespan=lifespan)

Server-Sent Events for token streaming #

SSE is the standard protocol for streaming LLM tokens to a chat UI. It's HTTP-based (no WebSocket upgrade), works through proxies and load balancers, and reconnects automatically on disconnect. The ALB timeout is the only constraint — set it to 120 s to handle long agent runs.

pythonsse_endpoint.py — streaming agent response via SSE
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
import asyncio, json

@app.post("/chat/stream")
async def chat_stream(request: ChatRequest):
    async def event_generator():
        async for chunk in graph.astream(
            {"messages": [{"role": "user", "content": request.message}]},
            stream_mode="messages",                    # stream individual tokens
        ):
            message_chunk, metadata = chunk
            if hasattr(message_chunk, "content") and message_chunk.content:
                # SSE format: "data: {json}\n\n"
                data = json.dumps({"type": "token", "content": message_chunk.content})
                yield f"data: {data}\n\n"

        yield 'data: {"type": "done"}\n\n'         # sentinel so client knows stream ended

    return StreamingResponse(
        event_generator(),
        media_type="text/event-stream",
        headers={
            "Cache-Control": "no-cache",
            "X-Accel-Buffering": "no",             # disable nginx/ALB buffering
            "Connection": "keep-alive",
        },
    )

WebSockets for bidirectional communication #

Use WebSockets when the user needs to interact mid-stream — approving a tool call, redirecting the agent, or providing additional context while the agent is running. SSE can't handle messages from client to server after the stream starts.

pythonws_endpoint.py — bidirectional agent interaction via WebSocket
from fastapi import WebSocket, WebSocketDisconnect

@app.websocket("/ws/agent/{session_id}")
async def agent_ws(ws: WebSocket, session_id: str):
    await ws.accept()
    config = {"configurable": {"thread_id": session_id}}
    try:
        while True:
            # Receive a message from the client
            data = await ws.receive_json()

            # Resume or start the graph, stream tokens back
            async for chunk in graph.astream(
                {"messages": [{"role": "user", "content": data["message"]}]},
                config=config,
                stream_mode="messages",
            ):
                message_chunk, metadata = chunk
                if hasattr(message_chunk, "content") and message_chunk.content:
                    await ws.send_json({"type": "token", "content": message_chunk.content})

            await ws.send_json({"type": "done"})

    except WebSocketDisconnect:
        pass   # client disconnected — checkpoint saves state, so session resumes later
✦
SSE vs WebSocket — the one-line rule If the user only reads tokens and occasionally submits a new message: SSE. If the user can interact with the agent while it's running (approve/deny tool calls, inject context): WebSocket. Start with SSE — it's simpler, more widely supported, and upgrades to WebSocket only if you have a concrete reason.
